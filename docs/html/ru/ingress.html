Ingress — Входная точка

Аутентификация, rate limiting, определение ServiceType, извлечение текста

RU EN

Назначение

Первая линия защиты. Валидирует токен, применяет rate limit, определяет ServiceType, извлекает TextParts для PII-сканирования, сохраняет RawBody.

Ingress — единственная точка входа для всех запросов к Prism. Все запросы проходят через аутентификацию и rate limiting до любой дальнейшей обработки.

Для нестандартных типов сервисов используется заголовок X-Prism-Service. Это позволяет поддерживать новые AI API без изменения маршрутизации по URL.

Извлечённые TextParts передаются в Privacy Pipeline для PII-сканирования. Оригинальное тело запроса сохраняется как json.RawMessage для pass-through к провайдеру.

Аутентификация

X-Prism-Token

Каждый запрос должен содержать заголовок X-Prism-Token. Ingress передаёт токен в Vault.ValidateToken() для проверки.

Проверка выполняется через HMAC(token) lookup в БД + constant-time compare. Без валидного токена запрос отклоняется с кодом 401 Unauthorized.

Rate Limiting

Token Bucket per projectID

Rate limiter реализован по алгоритму Token Bucket — отдельный bucket для каждого projectID. Параметр rate_limit_per_minute настраивается в конфигурации проекта.

При превышении лимита возвращается 429 Too Many Requests.

Безопасность

Prism биндится только на 127.0.0.1 (loopback). Это исключает доступ к API извне машины.

Plaintext токены не хранятся в БД — только HMAC хеши. Сам Ingress не имеет доступа к API-ключам провайдеров.
