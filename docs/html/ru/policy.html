Policy — Движок политик

Оркестратор: маршрутизация, бюджеты, failover

RU EN

Назначение

Единственный модуль, зависящий от всех остальных. Оркестрирует полный поток обработки запроса:

privacy → route → budget → cache → execute → restore → record

Все зависимости инжектируются через конструктор как интерфейсы. Никаких глобальных переменных.

Routing

Правила маршрутизации

Правила проверяются сверху вниз, первое совпавшее применяется. Router проверяет совместимость: выбранный провайдер должен поддерживать ServiceType запроса.

Условия (if) Описание
service_typeТип AI-сервиса (chat, image, embedding, tts, 3d_model, stt, moderation)
privacy_scoreОценка приватности из Privacy Pipeline
tagsТеги из заголовка X-Prism-Tags или auto-detected
project_idExplicit match по проекту
time_of_dayНапример, дешёвые модели ночью
Действия (then) Описание
providerЦелевой провайдер (Router проверяет SupportedServices())
modelOverride модели
fallbackПровайдер при недоступности основного
max_tokensОграничение ответа (для chat)

Валидация

Router.Validate()

Router.Validate() вызывается при загрузке routes.yaml и обнаруживает проблемы конфигурации при старте, а не при первом запросе:

  • Опечатки в именах провайдеров — все provider и fallback должны быть зарегистрированы в Registry
  • Несовместимость provider + service_type — провайдер не поддерживает указанный ServiceType
  • Unreachable rules — правила после catch-all (правило без if) никогда не сработают
  • Неизвестные service_types — опечатки в значениях service_type

Budget Guard

Проверка бюджетов до отправки

BudgetChecker использует EstimateCost() от провайдера для предварительной оценки стоимости. Порядок проверки — от конкретного к общему, срабатывает самый строгий:

Запрос (project=X, provider=Y)
    ↓
    estimate := provider.EstimateCost(req)
    ↓
    └─ проверить бюджет пары X × Y
    └─ проверить бюджет проекта X (все провайдеры)
    └─ проверить бюджет провайдера Y (все проекты)
    └─ проверить глобальный бюджет
    ↓
    все в рамках         → пропустить
    on_exceed=alert      → пропустить + уведомление
    on_exceed=downgrade  → переключить на дешёвую модель
    on_exceed=block      → 429 Too Many Requests

Для subscription-провайдеров дополнительно проверяются квоты периода — при исчерпании возвращается QuotaExceededError.

Failover

Retry + Fallback

Простая стратегия без exponential backoff:

primary → fail → 1 retry → fallback → fail → 503 Service Unavailable

При ошибке primary провайдера (timeout, 5xx) — один retry, затем переключение на fallback. Если оба провайдера недоступны — HTTP 503.

Зависимости

Deps struct

Engine принимает все зависимости через интерфейсы в конструкторе:

engine := policy.NewEngine(policy.Deps{
    Vault:        vaultImpl,        // vault.Vault
    Providers:    providerRegistry, // provider.Registry
    Privacy:      privacyPipeline,  // privacy.Pipeline
    BudgetCheck:  budgetChecker,    // policy.BudgetChecker
    AuditLog:     auditLogger,      // audit.Logger
    Cache:        semanticCache,    // cache.SemanticCache
    CostTracker:  costTracker,      // cost.Tracker
    RouterConfig: routesYAML,       // []byte (routes.yaml)
})

Пример routes.yaml

routes:
  - name: "sensitive_data"
    if:
      privacy_score: ">0.7"
    then:
      provider: ollama
      fallback: claude-haiku

  - name: "image_generation"
    if:
      service_type: image
    then:
      provider: openai            # DALL-E
      fallback: ollama             # Stable Diffusion

  - name: "embeddings_local"
    if:
      service_type: embedding
    then:
      provider: ollama             # nomic-embed-text
      fallback: openai

  - name: "cheap_tasks"
    if:
      service_type: chat
      tags: ["classify", "autocomplete"]
    then:
      provider: gemini-flash

  - name: "code_tasks"
    if:
      tags: ["code", "sql", "debug"]
    then:
      provider: claude-sonnet

  - name: "default"
    then:
      provider: claude-haiku

Интерфейс

type Router interface {
    // Route возвращает решение о маршрутизации.
    // Правила проверяются сверху вниз, применяется первое совпавшее.
    Route(ctx context.Context, req *types.SanitizedRequest) (*RoutingDecision, error)

    // Validate проверяет корректность routing rules при загрузке.
    Validate() []error
}

type BudgetChecker interface {
    // Check проверяет все применимые бюджеты.
    Check(ctx context.Context, projectID string, providerID types.ProviderID, estimate *types.CostEstimate) error
}

type Failover interface {
    // Execute пытается выполнить fn с primary провайдером.
    // При ошибке — один retry, затем fallback.
    Execute(ctx context.Context, primary, fallback types.ProviderID, fn func(types.ProviderID) error) error
}

type RoutingDecision struct {
    ProviderID    types.ProviderID
    ModelOverride string
    RuleName      string
    FallbackID    types.ProviderID
}
