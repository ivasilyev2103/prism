Privacy — Конвейер обфускации PII

Tiered PII detection: Regex → Ollama NER → Presidio

RU EN

Назначение

Обнаруживает PII в текстовых частях запросов и заменяет на placeholder'ы перед отправкой в облако. Обратимая обфускация — оригинальные данные восстанавливаются в ответе.

Работает только с TextParts, извлечёнными на уровне Ingress. Бинарные данные (изображения, аудио, 3D-модели) не сканируются.

Tiered Architecture

Трёхуровневая архитектура детекции

Tier Детектор Покрытие Зависимости
1 RegexDetector ~70% (email, phone, CC, SSN, IBAN, IP) Нет
2 + OllamaDetector ~95% (+имена, организации, адреса) Ollama
3 + PresidioDetector Максимальное Ollama + Python

Каждый следующий tier расширяет покрытие предыдущего. Tier 1 работает автономно без внешних зависимостей. Tier 2 добавляет NER через локальную LLM. Tier 3 подключает полнофункциональный NER-движок Presidio.

CompositeDetector

Объединяет результаты нескольких детекторов: union всех найденных сущностей, max score per overlap. Если два детектора нашли PII на перекрывающихся позициях, выбирается результат с наибольшей уверенностью.

CompositeDetector — основной детектор, используемый Pipeline. Конкретные детекторы (Regex, Ollama, Presidio) подключаются как плагины.

Substitution

Формат placeholder'ов

[TYPE_reqID8_N]

Где TYPE — тип PII (PERSON, EMAIL, ...), reqID8 — первые 8 символов ID запроса, N — порядковый номер.

Защита от prompt injection: при восстановлении заменяются только placeholder'ы, совпадающие с reqID8 текущего запроса. Чужие placeholder'ы остаются как есть.

Map Table

Хранение маппинга PII

Map Table (соответствие placeholder ↔ оригинальное значение) хранится только в RAM, в замыкании RestoreFunc.

Map Table никогда не передаётся между модулями и не сохраняется на диск. Уничтожается после первого вызова RestoreFunc — повторное восстановление невозможно.

Fail-closed vs Pass-through

ПровайдерРежимПоведение при ошибке детектора
Облачные (Claude, OpenAI, Gemini) fail-closed Запрос отклоняется. Данные не покидают машину.
Локальные (Ollama) pass-through Запрос проксируется без обфускации. Данные не покидают машину.

Для облачных провайдеров безопасность важнее доступности: если PII-детекция не сработала, запрос блокируется. Для локальных провайдеров данные остаются на машине в любом случае.

Presidio

Подключение Python sidecar

Presidio запускается как отдельный процесс (Python sidecar). Взаимодействие:

ПлатформаТранспортБезопасность
Unix/Linux/macOSUnix domain socketchmod 600
WindowsLoopback TCPTLS

Semaphore ограничивает количество параллельных запросов к Presidio для предотвращения перегрузки.

Профили

Уровни обфускации

ПрофильТипы PIImin_score
strict Все типы PII 0.5
moderate PERSON, EMAIL, CREDIT_CARD, SSN 0.85
off PII-сканирование отключено

Профиль strict обнаруживает максимум PII с низким порогом уверенности — возможны ложные срабатывания. Профиль moderate фокусируется на критичных типах данных с высоким порогом. Профиль off полностью отключает обфускацию.
