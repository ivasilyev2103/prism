Cost — Учёт расходов

Трекинг расходов, бюджеты, квоты подписок

RU EN

Назначение

Учёт расходов и потребления квот. Поддерживает many-to-many связь между проектами и провайдерами, 5 моделей биллинга и 4-уровневую иерархию бюджетов.

Использует in-memory write buffer с периодическим flush в SQLite для снижения write contention при concurrent нагрузке. Один провайдер может использовать разные BillingType для разных ServiceType — например, OpenAI: per_token для chat, per_image для DALL-E, per_request для moderation.

Модели биллинга

BillingType Как считается Примеры
per_token tokens × price/1M Claude, GPT
per_image images × price DALL-E
per_request фиксированная цена Moderation
per_second compute_time × price 3D generation
subscription квоты периода Claude Pro

Write Buffer

In-memory ring buffer

Record() добавляет RequestRecord в ring buffer. Фоновая горутина выполняет flush в SQLite пакетами:

  • Каждую 1 секунду (по таймеру)
  • Или при 100 записях в буфере (что наступит раньше)

Flush использует BEGIN IMMEDIATE + batch INSERT для атомарности. Graceful shutdown вызывает Flush() для принудительной записи буфера.

Record() → [ring buffer] → flush goroutine → BEGIN IMMEDIATE → batch INSERT → COMMIT
                                   ↑ timer (1s)
                                   ↑ buffer full (100)
                                   ↑ Flush() (shutdown)

При crash теряются записи из буфера (до 1 секунды или 100 записей). Это допустимо — метаданные, не финансовые транзакции.

Иерархия бюджетов

4-уровневая система

Проверяются все применимые бюджеты от конкретного к общему, срабатывает самый строгий:

Уровень Область Описание
1 project × provider Бюджет конкретной пары проект-провайдер
2 project Бюджет проекта (все провайдеры)
3 provider Бюджет провайдера (все проекты)
4 global Глобальный бюджет

Действия при превышении:

  • block — запрос отклоняется (HTTP 429)
  • downgrade_model — переключение на дешёвую модель
  • alert — запрос пропускается, отправляется уведомление

Subscription Tracking

Мониторинг квот за период

Для subscription-провайдеров Cost Tracker отслеживает потребление квот за текущий период. Квоты сбрасываются в sub_reset_day каждого месяца.

Тип квоты Описание
requests Количество запросов за период
input_tokens Входящие токены за период
output_tokens Исходящие токены за период
images Сгенерированные изображения за период

При исчерпании квоты возвращается QuotaExceededError. Лимит nil означает безлимитный доступ.

Интерфейс

type Tracker interface {
    // Record сохраняет метрики завершённого запроса (async-safe).
    Record(ctx context.Context, r *types.RequestRecord) error

    // Summary возвращает агрегированные данные за период.
    Summary(ctx context.Context, projectID string, providerID types.ProviderID, from, to time.Time) (*Summary, error)

    // QuotaUsage возвращает текущее потребление квот подписки.
    QuotaUsage(ctx context.Context, providerID types.ProviderID) (*QuotaUsage, error)

    // Flush принудительно записывает буфер в SQLite.
    Flush(ctx context.Context) error
}

type Summary struct {
    TotalUSD        float64
    RequestsCount   int64
    CacheSavingsUSD float64
    ByProvider      map[types.ProviderID]*ProviderSummary
    ByProject       map[string]*ProjectSummary
    ByPair          map[string]*PairSummary
}

type QuotaUsage struct {
    ProviderID        types.ProviderID
    PeriodStart       time.Time
    PeriodEnd         time.Time
    RequestsUsed      int64
    RequestsLimit     *int64
    InputTokensUsed   int64
    InputTokensLimit  *int64
    OutputTokensUsed  int64
    OutputTokensLimit *int64
    ImagesUsed        int64
    ImagesLimit       *int64
}
