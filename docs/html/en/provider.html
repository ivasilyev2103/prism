Provider — Provider Adapters

Normalizing different AI APIs to a unified Provider interface

RU EN

Purpose

Pass-through proxy to AI providers. Receives SanitizedBody and proxies to the provider API with minimal changes (auth header, endpoint rewrite).

Provider does not modify request content — it only adds authentication via Vault and constructs the correct URL endpoint.

Implementations

Provider ServiceTypes BillingType Status
Claude chat per_token Full
Ollama chat, embedding, image free (local) Full
OpenAI chat, image, embedding, tts, stt, moderation per_token, per_image, per_request, per_second Stub
Gemini chat, embedding per_token Stub

Execute()

Builds the provider URL endpoint, adds the auth header via Vault.SignRequest(), proxies SanitizedBody, and parses the response into types.Response.

The context must support cancellation (timeout). On provider error, an error is returned with the HTTP status and response body for diagnostics.

EstimateCost()

Pre-response cost estimate for the Budget Guard. Selects BillingType per ServiceType and request parameters.

For per_token, the estimate is based on input token count and model. For per_image — on the number of requested images. For Ollama, always returns zero cost.

SupportedServices()

Declares supported ServiceType values for Router compatibility checking. The Router uses this information to avoid sending image requests to a provider that only supports chat.

Interface

type Provider interface {
    ID() types.ProviderID
    SupportedServices() []types.ServiceType
    Execute(ctx context.Context, req *types.SanitizedRequest) (*types.Response, error)
    EstimateCost(req *types.ParsedRequest) (*types.CostEstimate, error)
    HealthCheck(ctx context.Context) error
}

type Registry interface {
    Get(id types.ProviderID) (Provider, error)
    GetForService(serviceType types.ServiceType) []Provider
    Register(p Provider)
    All() []Provider
}
