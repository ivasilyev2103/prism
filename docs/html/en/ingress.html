Ingress — Entry Point

Authentication, rate limiting, ServiceType detection, text extraction

RU EN

Purpose

First line of defense. Validates the token, applies rate limit, determines ServiceType, extracts TextParts for PII scanning, and preserves RawBody.

Ingress is the sole entry point for all requests to Prism. Every request passes through authentication and rate limiting before any further processing.

Extracted TextParts are passed to the Privacy Pipeline for PII scanning. The original request body is preserved as json.RawMessage for pass-through to the provider.

Authentication

X-Prism-Token

Every request must include the X-Prism-Token header. Ingress passes the token to Vault.ValidateToken() for verification.

Verification is performed via HMAC(token) lookup in the DB + constant-time compare. Without a valid token, the request is rejected with 401 Unauthorized.

Rate Limiting

Token Bucket per projectID

The rate limiter uses the Token Bucket algorithm — a separate bucket for each projectID. The rate_limit_per_minute parameter is configurable in the project configuration.

When the limit is exceeded, 429 Too Many Requests is returned.

Security

Prism binds only to 127.0.0.1 (loopback). This prevents API access from outside the machine.

Plaintext tokens are never stored in the DB — only HMAC hashes. Ingress itself has no access to provider API keys.
