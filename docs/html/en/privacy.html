Privacy — PII Obfuscation Pipeline

Tiered PII detection: Regex → Ollama NER → Presidio

RU EN

Purpose

Detects PII in text parts of requests and replaces them with placeholders before cloud transmission. Reversible obfuscation — original data is restored in the response.

Works only with TextParts extracted at the Ingress level. Binary data (images, audio, 3D models) is not scanned.

Tiered Architecture

Three-tier Detection Architecture

Tier Detector Coverage Dependencies
1 RegexDetector ~70% (email, phone, CC, SSN, IBAN, IP) None
2 + OllamaDetector ~95% (+names, organizations, addresses) Ollama
3 + PresidioDetector Maximum Ollama + Python

Each subsequent tier extends the coverage of the previous one. Tier 1 runs autonomously with no external dependencies. Tier 2 adds NER via a local LLM. Tier 3 connects the full-featured Presidio NER engine.

CompositeDetector

Merges results from multiple detectors: union of all found entities, max score per overlap. If two detectors find PII at overlapping positions, the result with the highest confidence is selected.

CompositeDetector is the primary detector used by the Pipeline. Specific detectors (Regex, Ollama, Presidio) are plugged in as backends.

Substitution

Placeholder Format

[TYPE_reqID8_N]

Where TYPE is the PII type (PERSON, EMAIL, ...), reqID8 is the first 8 characters of the request ID, and N is the sequence number.

Prompt injection protection: during restoration, only placeholders matching the current request's reqID8 are replaced. Foreign placeholders remain untouched.

Map Table

PII Mapping Storage

The Map Table (placeholder ↔ original value mapping) is stored in RAM only, inside the RestoreFunc closure.

The Map Table is never passed between modules and never persisted to disk. It is destroyed after the first call to RestoreFunc — repeated restoration is impossible.

Fail-closed vs Pass-through

ProviderModeBehavior on Detector Error
Cloud (Claude, OpenAI, Gemini) fail-closed Request is rejected. Data does not leave the machine.
Local (Ollama) pass-through Request is proxied without obfuscation. Data does not leave the machine.

For cloud providers, security takes priority over availability: if PII detection fails, the request is blocked. For local providers, data stays on the machine regardless.

Presidio

Python Sidecar Connection

Presidio runs as a separate process (Python sidecar). Communication:

PlatformTransportSecurity
Unix/Linux/macOSUnix domain socketchmod 600
WindowsLoopback TCPTLS

A semaphore limits the number of concurrent requests to Presidio to prevent overload.

Profiles

Obfuscation Levels

ProfilePII Typesmin_score
strict All PII types 0.5
moderate PERSON, EMAIL, CREDIT_CARD, SSN 0.85
off PII scanning disabled

The strict profile detects maximum PII with a low confidence threshold — false positives are possible. The moderate profile focuses on critical data types with a high threshold. The off profile disables obfuscation entirely.
