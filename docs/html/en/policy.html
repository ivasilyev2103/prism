Policy — Policy Engine

Orchestrator: routing, budgets, failover

RU EN

Purpose

The only module that depends on all others. Policy Engine orchestrates the full request processing pipeline:

privacy -> route -> budget -> cache -> execute -> restore -> record

All dependencies are injected via interfaces — no global state, no concrete type imports from other packages.

Routing

Rule-Based Provider Selection

Routing rules are defined in routes.yaml and checked top-to-bottom. The first matching rule is applied.

Conditions (all optional, combined with AND):

ConditionDescriptionExample
service_typeAI service categoryimage, chat, embedding
privacy_scorePII risk threshold>0.7, >=0.5
tagsRequest tags (any match)["code", "sql"]
project_idSource projectmy-app

Actions (applied when rule matches):

ActionDescription
providerTarget provider ID
modelOverride model (optional)
fallbackFallback provider on failure
max_tokensToken limit override

A rule without conditions (if omitted) acts as a catch-all default and must be the last rule.

Validation

Load-Time Checks

Router.Validate() is called at startup to detect configuration errors before any requests are processed:

  • Typos in provider names (provider not found in registry)
  • Provider + service_type incompatibility (provider does not support the service)
  • Unreachable rules (rules placed after a catch-all)
  • Unknown service_types (not in the known set)
  • Missing default route (no catch-all rule)

All errors are returned as a list, allowing operators to fix all issues at once.

Budget Guard

Pre-Send Budget Verification

Before sending any request to a provider, the engine checks budgets using the provider's EstimateCost() method. The 4-level budget hierarchy is evaluated:

global -> project -> provider -> project x provider pair

The strictest budget wins. If any applicable budget would be exceeded by the estimated cost, the request is blocked before it reaches the provider.

Failover

Simple Retry + Fallback

The failover strategy is deliberately simple:

primary -> fail -> 1 retry -> fallback provider
                                  |
                            both fail -> 503

There is no exponential backoff. If the primary provider fails twice and the fallback also fails, the request returns HTTP 503 (Service Unavailable).

Dependencies

Deps Struct

All dependencies are injected through the Deps struct. Every field is an interface — concrete implementations are provided at initialization:

type Deps struct {
    Privacy     privacy.Pipeline      // PII detection + sanitization
    Router      Router                // Rule-based provider selection
    BudgetCheck BudgetChecker         // 4-level budget hierarchy
    Providers   provider.Registry     // Provider registry
    CostTracker cost.Tracker          // Expense recording
    AuditLog    audit.Logger          // Append-only audit log
    Cache       cache.SemanticCache   // Semantic cache (may be nil)
    Failover    Failover              // Retry + fallback
}

Example routes.yaml

routes:
  - name: "sensitive_data"
    if:
      privacy_score: ">0.7"
    then:
      provider: ollama
      fallback: claude

  - name: "image_generation"
    if:
      service_type: image
    then:
      provider: openai
      fallback: ollama

  - name: "embeddings_local"
    if:
      service_type: embedding
    then:
      provider: ollama
      fallback: openai

  - name: "code_tasks"
    if:
      tags: ["code", "sql", "debug"]
    then:
      provider: claude

  - name: "default"
    then:
      provider: claude
      fallback: ollama

Interface

type Router interface {
    // Route returns a routing decision.
    // Rules are checked top-down, the first match is applied.
    Route(ctx context.Context, req *types.SanitizedRequest) (*RoutingDecision, error)

    // Validate checks correctness of routing rules at configuration load time.
    // Returns a list of errors: typos, unreachable rules, incompatible provider+service.
    Validate() []error
}

type BudgetChecker interface {
    // Check verifies all applicable budgets (global, project, provider, pair).
    // Returns BudgetExceededError or QuotaExceededError on exceeding.
    Check(ctx context.Context, projectID string, providerID types.ProviderID, estimate *types.CostEstimate) error
}

type Failover interface {
    // Execute tries to execute fn with the primary provider.
    // On error (timeout, 5xx) - one retry, then fallback provider.
    Execute(ctx context.Context, primary, fallback types.ProviderID, fn func(types.ProviderID) error) error
}

type RoutingDecision struct {
    ProviderID    types.ProviderID
    ModelOverride string            // "" = use from request
    RuleName      string            // name of the matched rule
    FallbackID    types.ProviderID
}
