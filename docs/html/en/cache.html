Cache — Semantic Cache

Response caching with VP-tree vector index

RU EN

Purpose

Caches responses to semantically similar requests, reducing cost and latency for repeated queries. Embeddings are generated locally via Ollama using the nomic-embed-text model — no data leaves the machine for cache operations.

A VP-tree (Vantage Point tree) provides O(log n) nearest-neighbor search over the embedding space.

Algorithm

Cache Lookup Flow

embed(text) -> cosine_similarity with cache
  -> similarity > 0.95 -> HIT  -> return cached response
  -> similarity < 0.95 -> MISS -> query provider -> store response

The threshold of 0.95 ensures only near-identical requests produce cache hits, preventing semantic drift from returning incorrect responses.

Encryption

Per-Entry AES-256-GCM

Cached responses are stored in sanitized form (PII replaced with placeholders). The PII mapping table that enables restoration is encrypted per-entry using AES-256-GCM.

PII never appears in plaintext on disk. Even if the cache database is compromised, the actual PII values remain encrypted.

VP-Tree

In-Memory Vector Index

The VP-tree (Vantage Point tree) is an in-memory spatial data structure optimized for nearest-neighbor search in metric spaces. It provides O(log n) lookup time for finding semantically similar cached entries.

SQLite serves as the persistent backend — on startup, all entry embeddings are loaded into the VP-tree. New entries are added to both the tree and SQLite simultaneously.

Degradation

Graceful Fallback

If Ollama is unavailable (down, unreachable, or slow):

  • Get() returns a cache miss — the request proceeds to the provider as normal
  • Set() is a no-op — the response is not cached

The cache never blocks or fails a request. It only improves performance when available.

Interface

type SemanticCache interface {
    // Get looks for a semantically similar request in the cache.
    // Returns (nil, nil) on miss or if caching is disabled for this ServiceType.
    Get(ctx context.Context, req *types.SanitizedRequest) (*types.Response, error)

    // Set saves a request and response in the cache.
    // For ServiceType with disabled caching - no-op.
    Set(ctx context.Context, req *types.SanitizedRequest, resp *types.Response) error

    // Invalidate removes entries by projectID (e.g., on privacy settings change).
    Invalidate(ctx context.Context, projectID string) error
}

type Embedder interface {
    // Embed returns an embedding for the text.
    // Implementation: nomic-embed-text via Ollama (locally).
    Embed(ctx context.Context, text string) ([]float32, error)

    // HealthCheck verifies Ollama availability.
    HealthCheck(ctx context.Context) error
}

type CachePolicy struct {
    ServiceType types.ServiceType
    Enabled     bool
    TTL         int // seconds; 0 = default
}
