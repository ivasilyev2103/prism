Cost — Expense Tracking

Cost tracking, budgets, subscription quotas

RU EN

Purpose

Tracks expenses and quota consumption across all AI service requests. Supports many-to-many project-provider relationships with 5 billing models and a 4-level budget hierarchy.

All cost data is stored in SQLite with an in-memory write buffer to reduce contention under concurrent load.

Write Buffer

In-memory Ring Buffer

Cost records are accumulated in an in-memory ring buffer to avoid per-request SQLite writes. The buffer is flushed to disk every 1 second or every 100 records, whichever comes first.

Flush uses BEGIN IMMEDIATE + batch INSERT for atomicity. On graceful shutdown, Flush() is called to persist all pending records before the process exits.

Budget Hierarchy

4-Level Budget System

Budgets are evaluated from most specific to most general. The strictest applicable budget wins:

LevelScopePriority
pairproject + provider1 (highest)
projectall providers for a project2
providerall projects for a provider3
globalentire system4 (lowest)

When a budget is exceeded, the configured action determines the response:

ActionBehavior
blockReturn HTTP 429 (Too Many Requests)
downgrade_modelSwitch to a cheaper model
alertAllow the request but emit a warning

Subscription Tracking

Quota Monitoring

For subscription-billed providers (e.g., Claude Pro), the cost module monitors quota usage per billing period. Tracked dimensions:

  • requests — total request count
  • input_tokens — tokens sent to the provider
  • output_tokens — tokens received from the provider
  • images — generated images count

Quotas reset on sub_reset_day of each month. When a quota is exhausted, QuotaExceededError is returned before sending the request.

Interface

type Tracker interface {
    // Record saves metrics of a completed request (async-safe).
    // Written to an in-memory buffer, flushed to SQLite in batches.
    Record(ctx context.Context, r *types.RequestRecord) error

    // Summary returns aggregated data for a period.
    // projectID and providerID are optional (empty string = all).
    Summary(ctx context.Context, projectID string, providerID types.ProviderID, from, to time.Time) (*Summary, error)

    // QuotaUsage returns current subscription quota consumption.
    QuotaUsage(ctx context.Context, providerID types.ProviderID) (*QuotaUsage, error)

    // Flush forcefully writes the buffer to SQLite (called on graceful shutdown).
    Flush(ctx context.Context) error
}

type Summary struct {
    TotalUSD        float64
    RequestsCount   int64
    CacheSavingsUSD float64
    ByProvider      map[types.ProviderID]*ProviderSummary
    ByProject       map[string]*ProjectSummary
    ByPair          map[string]*PairSummary // key: "projectID x providerID"
}

type QuotaUsage struct {
    ProviderID        types.ProviderID
    PeriodStart       time.Time
    PeriodEnd         time.Time
    RequestsUsed      int64
    RequestsLimit     *int64 // nil = unlimited
    InputTokensUsed   int64
    InputTokensLimit  *int64
    OutputTokensUsed  int64
    OutputTokensLimit *int64
    ImagesUsed        int64
    ImagesLimit       *int64
}
